설치

Reason comes by default in BuckleScript, a compiler that turns Reason code into JavaScript code.

필요조건: NPM(node와 함께) 혹은 Yarn.

BuckleScript & Reason을 전역으로 설치하려면 아래 명령을 실행하세요.

yarn global add bs-platform

(npm에서는 npm install -g bs-platform로 해요).

새 프로젝트

전역 설치했다면 간단한 프로젝트 생성기가 있을 겁니다. 아래 명령어를 써보세요.

bsb -init my-new-project -theme basic-reason

방금 만든 프로젝트를 컴파일하고 실행하려면 아래 명령어를 써요.

cd my-new-project
yarn build # npm에서는 npm run build로 해요.
node src/Demo.bs.js

That uses BuckleScript to compile Reason to JavaScript, then uses NodeJS to run the JavaScript. Feel free to use the generated JS files in whichever way you'd like, as if they're hand-written by you.

개발 환경에서는 컴파일을 하기 위해 매번 npm run build 명령을 사용하는 대신, 파일이 수정될 때마다 자동으로 다시 컴파일하도록 npm run start 명령을 사용하세요.

기본적으로 basic-reason 테마는 BuckleScript가 원본 Reason 파일과 같은 곳에 JavaScript 파일을 만들게 합니다. There's a reason why this is helpful. 다른 곳에 JavaScript 파일이 만들어지길 바란다면 bsconfig.json 파일의 in-source 항목을 false로 고치세요. 그러면 JavaScript 파일은 lib/js 디렉터리에 만들어질 겁니다.

또는 이 문서를 따라 ReasonReact 앱을 시작할 수 있습니다.

기존 프로젝트

우리에게 익숙한 명령을 통해 기존 프로젝트를 위한 여러 도구(toolchain)를 로컬로 설치할 수 있습니다.

yarn add --dev bs-platform

(npm에서는 npm install --save-dev bs-platform로 해요).

나머지는 위에서 설명한 것과 같습니다.
