What & Why

Reason とは？

Reason は新しい言語ではありません。百戦錬磨の言語、OCaml で動く新しい構文とツールチェインです。 Reason は OCaml に JavaScript プログラマーが慣れ親しんだ構文を与え、既存の NPM/Yarn のワークフローを使っているプログラマーに応えます。

つまり、Reasonは強く静的に型付けされた、より速く、よりシンプルなJavaScriptの親戚であり、負の遺産を取り除き、ES2030の機能を足し、JSとOCamlエコシステムの両方にアクセスできるものと考えることができます！

Reason は OCaml/Reason を相互運用し易いリーダブルな JavaScript にコンパイルする姉妹プロジェクトの BuckleScript の力を借りて JavaScript にコンパイルされます。 Reason は OCaml 自体のおかけでネイティブコードにもコンパイルされます。

なぜ Reason なのか？

「どうして新しい言語それ自体を学ばなければならないのか？」

Reasonに関して言えばそれは違います。

OCamlのコアランタイムセマンティクス(実行時の動作)は、そのままJavaScriptにマップされます。 JavaScriptのいくつかの部分を取り除き、いくつかの優れた機能を追加すれば、かなり読みやすいJSにコンパイルして、JavaScriptのエコシステムとツールの80%を直接使用することができます。 さらに、同じ言語を使ってベアボーンアセンブリ、iOS、Android、そしてマイクロコントローラ向けにさえもコンパイル出来ます！

しかし、高速なセマンティクスと堅牢な型システムを中心に設計された言語にフィットさせるために、JSのどの機能を再設計するかははっきりしていません。 しかし、パフォーマンスと安全性を中心にゼロから設計された言語から逆方向に作業し、慣れ親しんだWeb言語のより良いもののように見え、動作するように調整することが出来ます。

これらの決定は Reason の学習曲線を JS + 漸進的型システムを学ぶよりも緩やかにすることをもたらしました。その見返りとしてあなたが得られるのは:

  • 厳格で強い型システム。 Ocaml の型は 100% のカバレッジ（コード全行）、型推論（型は推論され手動で記述する必要はありません）、安全性（一度コンパイルされると、型が正しいことが保証されます）を持ちます。
  • シンプルさと実用性を重視。 Reasonでは、副作用、ミューテーション、オブジェクトを親しみやすく相互運用出来るようにしている一方で、言語の残りの部分は純粋で、イミュータブルかつファンクショナルに保っています。
  • パフォーマンス & サイズにフォーカス。 ReasonはビルドシステムであるbsbをJavaScriptにコンパイルするために使用し、それはインクリメンタルビルドを100ミリ秒未満で行います。 また、アウトプットも非常に小さくなります。
  • インクリメンタルな学習 & コードベース変換。 初日から完全に型付けされたファイルの恩恵を受けてください。 いざとなったら、Reason ファイルに生の JavaScript スニペットを貼り付けてください。
  • 素晴らしいエコシステム & ツール群。 お気に入りのエディタ、お気に入りの NPM パッケージ、そして、お気に入りの既存のスタックのあれやこれやそれのいずれかを使ってください。

* 信じられませんか？ JS - > Reason のチートシートをチェックするか、プレイグラウンド の Reason スニペットをいくつか試して、右側のアウトプットを観察してください。

なぜ後見言語として Ocaml なのか？なぜ [私のお気に入りの言語名が入る] ではないのか？

多くのバッキング言語は前節のポイントを満たすでしょう。しかし、以下の点は、OCaml言語の重要な特徴であり、Reasonの基盤として特に適しています。

  • ネイティブコードにコンパイルする機能。 OCaml のネイティブ（アセンブリ）の起動時間は、1桁ミリ秒です。 People are already starting to use Reason for native use cases today; meanwhile, we're focusing on adoption through great JavaScript compatibility.
  • 副作用、ミューテーション & 他の非常口。 これらは通常、言語の輝かしいセールスポイントではありません。しかし、緻密な相互運用/書き換えなしに、コードベース一部に向けて橋渡しできることが Facebook の私たちにとって重要です。 OCaml のデフォルトはイミュータブルで関数的なコードですが、非常口を持つことで、しばしば初期導入を単純にできます。
  • 実装の洗練が重要です。OCamlは20年以上かけて改良され、毎年改善されています。
  • React を書くための言語。 ReasonReact demonstrates how naturally React patterns play to the strengths of OCaml/Reason, and ReactJS itself was inspired by the functional, yet pragmatic philosophy of the ML family of languages (as described in the React To The Future talk by jordwalke).
  • 友好的で、成長しているコミュニティ。 The Reason community is welcoming of newcomers. The community has members from all over the world. Join the Discord channel and ask for help. Stick around to share what you've learned with other newcomers.

Reasonはあなたの求めていたものと違った？

ここでは、Reason/OCamlと似た概念/系統を共有しているものの、焦点が異なる代替案をいくつか紹介します。

  • OCaml. Reason is a syntax and toolchain built on top of the OCaml language. OCaml and Reason interoperate with eachother OCaml. You can compile plain OCaml with BuckleScript, and Js_of_ocaml can work with Reason as well.
  • Rust. Inspired by the ML family of langauges, but not garbage collected. Has excellent parallelism support.
  • Elm. Another great language in the ML family. Focuses on building web applications. See the widely praised talk on Elm, Let's Be Mainstream.
  • PureScript. Inspired by Haskell, compiles to the JavaScript.
  • Fable。OCaml に密接に関係している F# に基づいています。
  • ClojureScript. Dynamically typed language that prioritizes simplicity & great interop as well. Doesn't appear to have much in common with ReasonML. Shares some functional roots with the ML family of languages.
  • Swift. Language built by Apple, and interoperates with Objective-C well. Uses reference counted collection (but without automatic cycle breaking).
  • Haxe. Compiles to basically anything with a focus on game development.
