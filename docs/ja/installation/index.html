インストール

Reason は BuckleScriptというReasonのコードをJavaScriptのコードに変換するコンパイラに 標準で含まれています。

必要なもの: (nodeに付属する) NPMまたはYarn

BuckleScript と Reason をインストールするには:

yarn global add bs-platform

(または npm の場合 npm install -g bs-platform)

新規プロジェクト

グローバルにインストールしたパッケージには、シンプルなプロジェクト作成ツールが含まれます。次のコマンドを試してみましょう。

bsb -init my-new-project -theme basic-reason

作成したプロジェクトをコンパイルおよび実行するには次のコマンドを実行します:

cd my-new-project
yarn build # or npm run build, for npm
node src/Demo.bs.js

ReasonはBuckleScriptによってJavaScriptにコンパイルされたのちに、NodeJSを使ってJavaScriptコードが実行されます。生成されたJSファイルは、直接記述した場合と同じように、好きな方法で利用することができます。

開発の際は、npm run build を都度実行してコンパイルする代わりに、npm run start を実行することによって、ファイル変更のたびに自動的に再コンパイルするウォッチャーを起動することができます。

デフォルトではbasic-reasonテーマの設定によって、JSファイルは生成元のReasonファイルと同じディレクトリに並んで生成された出力されます。 これには理由があります。 もし生成するファイルを異なる場所に出力したい場合は、bsconfig.jsonを編集し、in-source を falseに設定します。このように設定することによって、JSファイルの出力先はlib/jsディレクトリに変更されます。

もしくは、 ReasonReact app を始めるには、こちらの説明を参照してください。

既存のプロジェクト

いつものコマンドを使って、既存のプロジェクトだけにツールチェインをインストール出来ます。

yarn add --dev bs-platform

(または npm の場合 npm install --save-dev bs-platform)

あとは新規プロジェクトの場合と同じです。
