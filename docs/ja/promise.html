Promise

Reason は BuckleScript による JavaScript の promisesをビルトインサポートしています。一般的に必要な3つの関数は次のとおりです。

  • Js.Promise.resolve: 'a => Js.Promise.t('a)
  • Js.Promise.then_: ('a => Js.Promise.t('b), Js.Promise.t('a)) => Js.Promise.t('b)
  • Js.Promise.catch: (Js.Promise.error => Js.Promise.t('a), Js.Promise.t('a)) => Js.Promise.t('a)

加えて、Reason 側で JS の promise を作成する方法は次のとおりです。

  • Js.Promise.make: ( ( ~resolve: (. 'a) => unit, ~reject: (. exn) => unit ) => unit ) => Js.Promise.t('a)

この型シグネチャは make が、2つの名前付き引数、resolve と reject を受け取るコールバックを、受け取ることを意味します。 両引数はそれ自身がカリー化されていないコールバック（それらにはドットがある）です。 make は作成した promise を返します。

使い方

パイプ演算子（Pervasives moduleの "Composition operators" を参照）を使用します:

let myPromise = Js.Promise.make((~resolve, ~reject) => resolve(. 2));

myPromise
|> Js.Promise.then_(value => {
     Js.log(value);
     Js.Promise.resolve(value + 2);
   })
|> Js.Promise.then_(value => {
     Js.log(value);
     Js.Promise.resolve(value + 3);
   })
|> Js.Promise.catch(err => {
     Js.log2("Failure!!", err);
     Js.Promise.resolve(-2);
   });

備考: 私達は将来、JS promises (async/await) のための専用の構文を提供するかもしれません。乞うご期待！
